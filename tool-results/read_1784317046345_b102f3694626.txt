   272тЖТ   301тЖТ   301тЖТ   301тЖТ### 2.2 Design rules for the profile
   273тЖТ   302тЖТ   302тЖТ   302тЖТ
   274тЖТ   303тЖТ   303тЖТ   303тЖТ1. **Every downstream engine consumes this object.** The Eligibility Engine, Financial Engine, DPR Engine, and PDF Engine all receive (a subset of) the `ProjectProfile` as their primary input. None reads conversation history, form state, or partial data.
   275тЖТ   304тЖТ   304тЖТ   304тЖТ
   276тЖТ   305тЖТ   305тЖТ   305тЖТ2. **Money is never stored as floating point.** All rupee amounts are integers (whole rupees or paise тАФ decided at implementation, recorded in [05-financial-engine.md](05-financial-engine.md)). The Financial Engine consumes integer values and returns integer values.
   277тЖТ   306тЖТ   306тЖТ   306тЖТ
   278тЖТ   307тЖТ   307тЖТ   307тЖТ3. **The profile is the output of the interview, not the interview itself.** Conversation messages, UI state, and the question-selection logic live in `features/ai/` тАФ they are not part of the profile.
   279тЖТ   308тЖТ   308тЖТ   308тЖТ
   280тЖТ   309тЖТ   309тЖТ   309тЖТ4. **Optional fields (`?`) vs mandatory fields.** The Validation Engine defines which fields are mandatory for the profile to be considered complete. The schema itself allows `?` for fields that may be unknown or inapplicable тАФ the Validation Engine enforces the business rules.
   281тЖТ   310тЖТ   310тЖТ   310тЖТ
   282тЖТ   311тЖТ   311тЖТ   311тЖТ5. **The profile is immutable during a single engine run.** An engine receives a snapshot of the profile; it does not observe mid-interview mutations.
   283тЖТ   312тЖТ   312тЖТ   312тЖТ
   284тЖТ   313тЖТ   313тЖТ   313тЖТ6. **Both interaction models produce the same type.** There is no `AiProjectProfile` vs `FormProjectProfile`. The Zod schema that validates the AI interview output is the same Zod schema that validates the guided forms.
   285тЖТ   314тЖТ   314тЖТ   314тЖТ
   286тЖТ   315тЖТ   315тЖТ   315тЖТ### 2.3 Field ownership
   287тЖТ   316тЖТ   316тЖТ   316тЖТ

   290тЖТ   319тЖТ   319тЖТ   319тЖТThis creates a powerful invariant: **no subsystem silently overwrites another subsystem's data.**

   320тЖТ   349тЖТ   349тЖТ   349тЖТ## 3. The AI Reasoning Pipeline
   321тЖТ   350тЖТ   350тЖТ   350тЖТ
   322тЖТ   351тЖТ   351тЖТ   351тЖТEvery user message during the AI interview triggers a **deterministic, multi-stage reasoning pipeline** inside `features/ai/`. This pipeline is **orchestration code** тАФ it lives in the feature layer, not in any engine, not in the AI provider. The AI model is called at specific stages; the rest is deterministic logic.
   323тЖТ   352тЖТ   352тЖТ   352тЖТ
   324тЖТ   353тЖТ   353тЖТ   353тЖТ### 3.1 Pipeline stages
   325тЖТ   354тЖТ   354тЖТ   354тЖТ
   326тЖТ   355тЖТ   355тЖТ   355тЖТ```
   327тЖТ   356тЖТ   356тЖТ   356тЖТUser message received
   328тЖТ   357тЖТ   357тЖТ   357тЖТ  тФВ
   329тЖТ   358тЖТ   358тЖТ   358тЖТ  тЦ╝
   330тЖТ   359тЖТ   359тЖТ   359тЖТтФМтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФР
   331тЖТ   360тЖТ   360тЖТ   360тЖТтФВ STAGE 1 тАФ Response Parsing                                       тФВ
   332тЖТ   361тЖТ   361тЖТ   361тЖТтФВ                                                                  тФВ
   333тЖТ   362тЖТ   362тЖТ   362тЖТтФВ Parse the raw user message into a structured intermediate         тФВ
   334тЖТ   363тЖТ   363тЖТ   363тЖТтФВ representation. This is NOT field extraction тАФ it is parsing the  тФВ
   335тЖТ   364тЖТ   364тЖТ   364тЖТтФВ user's intent and content into a form the next stages can        тФВ
   336тЖТ   365тЖТ   365тЖТ   365тЖТтФВ process.                                                         тФВ
   337тЖТ   366тЖТ   366тЖТ   366тЖТтФВ                                                                  тФВ
   338тЖТ   367тЖТ   367тЖТ   367тЖТтФВ Responsibilities:                                                тФВ
   339тЖТ   368тЖТ   368тЖТ   368тЖТтФВ тАв Detect if the user is answering a question, asking a           тФВ
   340тЖТ   369тЖТ   369тЖТ   369тЖТтФВ   question, correcting a previous answer, or requesting          тФВ
